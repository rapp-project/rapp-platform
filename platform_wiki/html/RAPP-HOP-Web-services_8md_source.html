69 ## Service specifications - Request arguments and response objects
70 
71 
72 The Web Services listen to **POST** requests and can parse contents of the following type (Content-Type):
73 
74 - application/x-www-form-urlencoded
75 - multipart/form-data
76 
77 A lot of services require from the client to upload a file to the server for processing, like Computer-VIsion, Speech-Recognition and more.
78 The *multipart/form-data* content-type is used to call a service that requresres file upload to the server.
79 Otherwise, use *application/x-www-form-urlencoded*.
80 
81 All data, except files, have to be be send under a field named **json**.
82 In case of using *application/x-www-form-urlencoded* type this will look like:
83 
84 ```http
85 POST /hop/ontology_subclasses_of HTTP/1.1
86 Connection: keep-alive
87 ...
88 Content-Type: application/x-www-form-urlencoded
89 
90 json=%7B%22query%22%3A+%22Oven%22%7D
91 ```
92 
93 and in case of multipart/form-data:
94 
95 ```http
96 POST /hop/face_detection HTTP/1.1
97 Connection: keep-alive
98 ...
99 Content-Type: multipart/form-data; boundary=993ac36c568042aa86582023b7422092
100 
101 --993ac36c568042aa86582023b7422092
102 Content-Disposition: form-data; name="json"
103 
104 {"fast": false}
105 --993ac36c568042aa86582023b7422092
106 Content-Disposition: form-data; name="file"; filename="Lenna.jpg"
107 
108 <file-data-here>
109 ...
110 
111 ```

115 -------------------------------
116 
117 #### Face-Detection
118 
119 ##### Service-Url
120 
121 ```
122 /hop/face_detection
123 ```
124 
125 ##### Service request arguments
126 
127 ```js
128 { file: '', fast: false }
129 ```
130 
131 - **file**: Path to the uploaded file (**image file**), stored by hop-server. This is the form-data name to attach the file to.
132 - **fast** (Boolean): If true, detection will take less time but it will be less accurate.
133 
134 ##### Service response
135 
136 application/json response.
137 
138 ```js
139 { faces: [{<face_1>}, ..., {<face_n>}], error: '' }
140 ```
141 
142 - **faces** (Array): Vector with the recognized faces in an image frame.
143 - **error** (String): Error message, if one occures.
144 
145 where **face_x** is an object of the following structure:
146 
147 ```js
148 face: { up_left_point: {<point>}, down_right_point: {<point>} }
149 ```
150 
151 - **up_left_point** (Object): The up-left point coordinates of the detected face.
152 - **down_right_point** (Object): The down-right point coordinates of the detected face.
153 
154 and **point** coordinates are presented in Cartesian Coordinate System as:
155 
156 ```js
157 point: { x: <value_int>, y: <value_int> }
158 ```
159 
160 Response Sample:
161 
162 ```javascript
163  {
164  faces: [{
165  up_left_point: { y: 200, x: 212 },
166  down_right_point: { y: 379, x: 391 }
167  }],
168  error: ''
169  }
170 ```

554 ----------------------------------
555 
556 #### Cognitive-Test-Selector
557 
558 ##### Service-Url
559 
560 ```
561 /hop/cognitive_test_chooser
562 ```
563 
564 ##### Service request arguments
565 
566 ```js
567 { test_type: '', test_subtype: '', test_diff: '', test_index: '' }
568 ```
569 
570 - **test_type** (String): Cognitive Exercise test type. Can be one of
571  * 'ArithmeticCts'
572  * 'AwarenessCts'
573  * 'ReasoningCts'
574  * ''
575 - **test_subtype** (String): Use this to force select from this subtype. Defaults to empty string "".
576 - **test_diff** (String): Use this to force select from this difficulty. Defaults to empty string "".
577 - **test_index** (String): Use this to force select from this id. Defaults to empty string "".
578 
579 i.e
580 
581 ```js
582 { test_type: 'Arithmetic', test_subtype: 'BasicArithmetic', test_diff: '1', test_index: '1' }
583 ```
584 
585 For more information on available exercises have a look [here](https://github.com/rapp-project/rapp-platform/tree/master/rapp_cognitive_exercise)
586 
587 ##### Service response
588 
589 application/json response.
590 
591 ```javascript
592  { questions: [], possib_ans: [], correct_ans: [], test_instance: '', test_type: '', test_subtype: '', error: '' }
593 ```
594 
595 - **questions** (Array): The exercise set of questions.
596 - **possib_ans**(Array): The set of answers for each question. vector<vector<string>>
597 - **correct_ans**(Array): The set of correct answers for each question. vector<string>
598 - **test_instance** (String): Returned test name. For example, 'ArithmeticCts_askw0Snwk'
599 - **test_type** (String): Cognitive exercise class/type. Documentation on Cognitive Exercise classes can be found [here](https://github.com/rapp-project/rapp-platform/tree/CognitiveSystem/rapp_cognitive_exercise)
600 - **test_subtype** (String): Cognitive exercise sub-type. Documentation on Subtypes can be found [here](https://github.com/rapp-project/rapp-platform/tree/CognitiveSystem/rapp_cognitive_exercise)
601 - **error** (String): Error message, if one occures.

866 
867 ##### Service-Url

872 
873 ##### Service request arguments
874 
875 ```js

877 ```
878 
879 - **map_name** (String): The map name to use.
880 - **robot_type** (String): The robot type. It is required to determine it's parameters (footprint etc.)
881 - **algorithm** {String}: The path planning algorithm to apply.
882 - **start** {Object}: Start pose of the robot. (ROS-GeometryMsgs/PoseStamped)
883 - **goal**: Goal pose of the robot. (ROS-GeometryMsgs/PoseStamped)
884 
885 
886 More information on argument values under the [rapp_path_planning package](https://github.com/rapp-project/rapp-platform/tree/master/rapp_path_planning/rapp_path_planning)
887 
888 ##### Service response
889 
890 ```js
891 { plan_found: 0, path: [], error: '' }
892 ```
893 
894 - **plan_found** (String): Plan Status. Can be one of
895  * 0 : path cannot be planned.
896  * 1 : path found.
897  * 2 : wrong map name.
898  * 3 : wrong robot type.
899  * 4 : wrong algorithm.
900 - **path**: if plan_found is true, this is an array of waypoints from start to goal, where the first one equals start and the last one equals goal.
901 - **error** (String): Error message, if one occures.
