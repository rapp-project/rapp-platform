Go to the documentation of this file.
1 It is recommended to study on [Rapp-Testing-Tools](https://github.com/rapp-project/rapp-platform/wiki/RAPP-Testing-Tools) first.
2 
3 Basic documentation on "**Developing Integration Tests**" can be found [here](https://github.com/rapp-project/rapp-platform/tree/master/rapp_testing_tools).
4 
5 A more-in-depth information on how to write your first integration test is presented here.
6 
7 The [template.py](https://github.com/rapp-project/rapp-platform/blob/master/rapp_testing_tools/scripts/default_tests/template.py) will be used as a reference.
8 
9 Each integration test must have the following characteristics:
10 
11  - Each test is written as a seperate python source file (.py).
12  - Each test is implemented as a RappInterfaceTest class.
13  - The RappInterfaceTest class must define two member methods:
14  - execute()
15  - validate()
16  - The Python RAPP-API is imported into our code in order to invoke Platform. Web Service requests.
17 
18 
19 So lets first create our RappInterfaceTest class which includes the aforementioned member methods.
20 
21 ```python
22 class RappInterfaceTest:
23  """ Integration test class """
24  def __init__(self):
25  pass
26 
27  def execute(self):
28  pass
29 
30  def validate(self):
31  pass
32 ```
33 
34 Next we need to include the Python RAPP-API module and create an in instance of the RappCloud class which will allow us to invoke Platform Web Service calls.
35 
36 ```python
37 from RappCloud import RappCloud
38 
39 class RappInterfaceTest:
40  """ Integration test class """
41  def __init__(self):
42  self.rappCloud = RappCloud()
43  pass
44 
45 ```
46 
47 Load any files required for integration test on initialization of the RappInterfaceTest instance and declare the validation values.
48 
49 ```python
50  def __init__(self):
51  self.rappCloud = RappCloud()
52 
53  # Set the file path
54  self.file_uri = 'PATH_TO_LOADED_FILE'
55  # Set the valid results.
56  self.valid_results = {}
57 ```
58 
59 Next we need to implement the validation function, **validate(self)**.
60 
61 ```python
62  def validate(self, response):
63  error = response['error']
64  if error != "":
65  return [error, self.elapsed_time]
66 
67  return_data = response['qr_centers']
68  # Check if the returned data are equal to the expected
69  if self.valid_results == return_data:
70  return [True, self.elapsed_time]
71  else:
72  return ["Unexpected result : " + str(return_data), self.elapsed_time]
73 ```
74 
75 Notice that the validate() function has a parameter named response. This is the response object returned from the Platform Service call.
76 It is critical to return validation results as presented above!!
77 
78 ```python
79 return [True, self.elapsed_time]
80 ```
81 
82 If the response was succesfully validated,
83 
84 ```python
85 return ["Unexpected result : " + str(return_data), self.elapsed_time]
86 ```
87 
88 if the validation failed.
89 
90 
91 The [timeit](https://docs.python.org/2/library/timeit.html) python module is used to time the execution of the test.
92 **This will be raplaced in v0.6.0 release and execution time of each test will be calculated by the rapp-testing-core-engine.**
93 
94 Implement the execution function. The execution function is called by the test-core-engine to execute implemented test.
95 The execution function must:
96 
97 - Call the, **to-test**, Platform Web Service through the Python RAPP-API call.
98 - Invoke the validation function.
99 
100 ```python
101 def execute(self):
102  start_time = timeit.default_timer()
103  # Call the Python RappCloud service
104  response = self.rappCloud.qr_detection(self.file_uri)
105  end_time = timeit.default_timer()
106  self.elapsed_time = end_time - start_time
107  return self.validate(response)
108 ```
109 
110 
111 You will notice that the above implementation of the execute() method calls the qr_detection Service!!
112 
113 ```python
114 response = self.rappCloud.qr_detection(self.file_uri)
115 ```
